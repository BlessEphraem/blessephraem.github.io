# 🎬 JSX Runner — UXP Plugin for Adobe Premiere Pro

> A lightweight plugin that lets you **browse, select, and execute your own UXP/JSX scripts** directly inside Premiere Pro — no copy-paste, no external tools.

[![GitHub](https://img.shields.io/badge/GitHub-JSX_Runner-181717?style=for-the-badge&logo=github)](https://github.com/BlessEphraem/JSX-Runner)

---

## ✨ Features

- 📁 **Folder browser** — point the plugin to any local folder containing your `.jsx` or `.js` scripts
- 📋 **Script list** — all scripts are listed; single click to select, double-click to run
- ▶️ **One-click execution** — runs the selected script inside Premiere Pro's UXP runtime with full access to the `premierepro` API
- 🐛 **Built-in debug console** — optional visual console that intercepts `console.log`, `console.warn`, and `console.error` output in real time

---

## 📋 Prerequisites

- **Adobe Premiere Pro 25.6.1 or later**, or a **Beta version** with UXP plugin support enabled.
- **Adobe Creative Cloud** must be installed and running on your machine.

---

## 🚀 Installation

1. Download the latest [release](https://github.com/BlessEphraem/JSX-Runner/releases).

   > ⚠️ **Note on cloning:** Download the release instead of cloning the repository. If you want to edit the plugin, download the release and import it directly into the **Adobe UXP Developer Tool**.

2. **Do not move any files.** The `.bat` installer must remain in the same folder as the `.ccx` plugin file.

3. Right-click the `.bat` file and run it **as Administrator**.

The script will automatically locate the Adobe plugin installer and register the plugin with Premiere Pro.

---

## 🛠️ Usage

1. Open the plugin panel inside Premiere Pro
2. Go to **Settings**, click **Browse** to select the folder where your `.jsx` scripts are stored
3. Click a script in the list to **select** it
4. Click **Run** (or double-click the script) to **execute** it
5. Optionally, enable the **Debug Console** in Settings to see real-time log output
